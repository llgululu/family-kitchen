# 情侣厨房 —— 上线 Checklist

> 把 [部署文档](./07-deployment.md) 里的步骤拆成时间表，按顺序勾选。
> ⏱️ 关键路径：**ICP 备案最长**，提前 3-4 周开始；其余按 T-7 倒推。

---

## ⏰ T-30 天（最早可以开始的事）

### 主体 & 合规

- [ ] 确认小程序主体（个人 / 企业）
- [ ] 准备 ICP 备案材料：身份证、域名证书、服务器订单截图、备案号变更承诺函等
- [ ] **提交 ICP 备案** ← 最长 20 个工作日，最先做

### 基础采购

- [ ] 购买云服务器（推荐腾讯云，与微信生态同厂）
- [ ] 购买域名（不要用 .gov / .edu / 含敏感词）
- [ ] 开通对象存储桶 COS / OSS
- [ ] 注册 Sentry 账号并建 project（可选）

---

## T-14 天（备案审核中，开始技术准备）

### 服务器初始化

- [ ] 系统更新 + 时区 + 防火墙（仅开 22 / 80 / 443）
- [ ] 安装 Node 20 + pnpm + Nginx + Docker
- [ ] 创建非 root 部署账号
- [ ] 配置 SSH key（禁用密码登录）

### 微信公众平台准备

- [ ] **轮换 AppSecret**（开发期暴露过的）
- [ ] 服务类目确认无误
- [ ] 在「用户隐私保护指引」里登记将使用的接口和用途
- [ ] 准备隐私政策 / 用户协议页面文案

### 代码准备

- [ ] 主分支锁定，建 `release/v0.1.0` 分支
- [ ] 跑一遍全量测试：`pnpm -r run test`
- [ ] 全量 lint：`pnpm -r run lint`
- [ ] 数据库 schema 冻结，检查所有迁移文件

---

## T-7 天（备案应当下来了）

### 域名 + HTTPS

- [ ] 域名 DNS：`api.xxx.com` 和 `admin.xxx.com` 都指向服务器 IP
- [ ] Let's Encrypt 申请证书
- [ ] Nginx 配置生效 + HTTPS 跳转
- [ ] 验证：`curl -I https://api.xxx.com` 返回 200

### 微信小程序服务器配置

- [ ] 公众平台 → 服务器域名 → 添加 `https://api.xxx.com`（request）
- [ ] 添加 uploadFile 域名（同上）
- [ ] 添加 downloadFile 域名（COS）

### 订阅消息模板

- [ ] 在公共模板库挑选并申请 4 个模板
  - [ ] 订单接单提醒
  - [ ] 订单完成提醒
  - [ ] 催菜提醒
  - [ ] 成就解锁
- [ ] 把 4 个 `tmpl_id` 填到生产 `backend/.env` 的 `WX_TMPL_*`

### 生产环境部署

- [ ] MySQL 容器跑起来 + root 密码改为强密码
- [ ] 创建 `family_kitchen` 数据库
- [ ] `backend/.env` 填好所有变量（**重新生成 JWT_SECRET**）
- [ ] 跑数据库迁移：`pnpm --filter ... run db:migrate:deploy`
- [ ] 构建并 PM2 启动 backend
- [ ] curl 健康检查通过
- [ ] admin 构建 + 上传到 `/var/www/admin`
- [ ] 浏览器打开 https://admin.xxx.com 能登录

### 对象存储

- [ ] COS 桶创建完毕、设置公有读
- [ ] CORS 规则配置：允许 mini-app + admin 域名 GET
- [ ] 上传一个测试图，浏览器能打开 URL

---

## T-3 天（最后冲刺）

### 数据准备

- [ ] 用 admin 账号登录，创建一个测试家庭 + 几道菜谱
- [ ] 走一遍完整流程（点菜 → 接单 → 上菜 → 评价 → 看时间线）
- [ ] 校验：爱心币结算正确、成就触发、时间线生成

### 小程序构建上传

- [ ] `mini-app/src/manifest.json` 版本号 `0.1.0` → `1.0.0`
- [ ] 把 `VITE_API_BASE_URL` 设为 `https://api.xxx.com/api/v1`
- [ ] `pnpm --filter @family-kitchen/mini-app run build:mp-weixin`
- [ ] 微信开发者工具：体验版上传 + 内部测试
- [ ] 内部测试 3-5 个真实账号：登录、家庭、点菜、评价、所有 Tab

### 监控 & 告警

- [ ] Sentry 接收到测试错误（手动触发一个）
- [ ] UptimeRobot / 云监控配置 `/api/v1/health/db` 探针
- [ ] 备份 cron 配置好：`mysqldump` 每日 2:00
- [ ] PM2 logrotate 安装

---

## T-1 天（上线前一天）

### 最终回归

- [ ] **再走一遍完整流程**（端到端）
- [ ] 异常场景：
  - [ ] 没加入家庭时各 Tab 引导正确
  - [ ] 重复邀请码加入会被拒
  - [ ] 一家庭多单会被拒
  - [ ] 评价后能看到爱心币变动
- [ ] 微信开发者工具：体验版扫码再次试用
- [ ] 检查所有 `console.log` / `// TODO` / `// 占位` 是否需要清理
- [ ] backend 日志无频繁 ERROR

### 提交审核

- [ ] 微信公众平台 → 版本管理 → 提交审核
- [ ] 审核备注里说明：测试账号、核心流程截图、隐私使用场景

⏱️ 审核通常 1-3 天，节假日更久。

---

## D-Day（审核通过日）

### 发布

- [ ] 公众平台点「发布」
- [ ] 截图微信版本号 / 时间，作为发布记录
- [ ] 在 Sentry / 监控 Dashboard 加一个 release 标签

### 灰度（可选）

- [ ] 微信支持的话用「灰度发布」，先 10% → 50% → 100%
- [ ] 每个阶段观察 30 分钟：错误率 / DAU / 接口耗时

### 周知

- [ ] 写一个对外的 onboarding 文档（怎么注册、怎么用）
- [ ] 准备一份首批用户群发文案（邀请测试）

---

## D+1 ~ D+7（监控期）

### 每日检查

- [ ] Sentry 错误数 < 上线前 110%
- [ ] 接口平均响应时间 < 500ms
- [ ] PM2 进程无频繁重启
- [ ] MySQL 慢查询日志
- [ ] 数据库连接数 / 磁盘空间趋势
- [ ] COS 流量是否突增

### 数据健康

- [ ] DAU / WAU 上升曲线正常
- [ ] 注册转化率（登录 → 创建家庭 → 第一单）
- [ ] 双活率（北极星指标）

### 异常预案

- [ ] 503 / 502 → PM2 重启 / Nginx 重启
- [ ] 数据库连不上 → 检查 docker ps + 磁盘
- [ ] 大量 401 → 排查 token 过期 / clock skew
- [ ] 上传图片失败 → COS keys 是否被禁

---

## D+30 之后

### 复盘

- [ ] 数据：DAU、订单完成率、双活率（PRD §7）
- [ ] Sentry：Top 5 错误是否已修
- [ ] 用户反馈：admin 看 `/admin/feedback`
- [ ] 性能：哪些接口需要优化

### 第一波迭代

- [ ] 收集到的高频反馈进入 backlog
- [ ] 准备 v1.1（参考 PRD §9 演进路径）
