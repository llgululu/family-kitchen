## 项目目录约定

```
family-kitchen/
├── admin/        # 管理后台（Vue 3 + Element Plus + TypeScript）
├── backend/      # 后端服务（NestJS + Prisma + MySQL）
├── mini-app/     # 微信小程序前端（uni-app + Vue 3）
├── packages/     # 跨端共享（枚举、常量）
└── docs/         # 产品文档（本目录）
```

各模块的技术文档（架构设计、API 规范、部署说明）应放在对应目录的 `README.md` 中，本目录仅承载产品视角文档。

## 关键产品决策（一页速览）

| 决策点   | 选择                                          |
| -------- | --------------------------------------------- |
| 平台     | 微信小程序                                    |
| 核心场景 | 家庭内部点菜（一方做、一方点）                |
| 账号模型 | 多成员家庭空间，MVP 仅支持双人                |
| 互动玩法 | 即时沟通 + 爱心币 + 成就 + 食物日记，四位一体 |
| 商业化   | MVP 不做，不接外部商家                        |
| 数据隐私 | 家庭内严格隔离，不做对外社交                  |

## 技术栈一页速览

| 维度     | 选型                                                   |
| -------- | ------------------------------------------------------ |
| 仓库结构 | monorepo + pnpm workspace                              |
| 后端     | NestJS + TypeScript + Prisma + MySQL 8.x               |
| 小程序   | uni-app + Vue 3 + JavaScript                           |
| 管理后台 | Vue 3 + Element Plus + TypeScript                      |
| API 契约 | OpenAPI / Swagger（自动生成类型至 `packages/shared/`） |
| 认证     | wx.login + JWT                                         |
| 实时通信 | WebSocket + 微信订阅消息                                |
| 对象存储 | MinIO（开发期）→ COS/OSS（生产期）                     |
| CI       | GitHub Actions（lint + test，不部署）                  |
| 监控     | Sentry（三端）+ pino 日志                              |

详细技术决策与替代方案表见 [06-tech-stack.md](./06-tech-stack.md)。

## 已确认的关键设计决策

> MVP 阶段已对原先所有待定项做了明确取舍。如需后续回顾，集中在以下两处：

- **产品策略级**：[PRD §8 关键设计决策](./01-prd-overview.md#8-关键设计决策)
  - 爱心币不设封顶、不做衰减（30 天后复盘）
  - 家庭空间解散：单方可退出 + 定时任务清理
  - 单家庭同时只允许 1 单活跃

- **实现细节级**：[数据模型 §6 已确认设计细节](./05-data-model.md#6-已确认设计细节)
  - 催菜消息使用独立类型 `rush`，便于服务端限频
  - 徽章定义使用 BadgeDefinition 数据库表，管理员可动态管理
  - 多家庭时代（P2）爱心币留原家庭、个人成就跟用户走
