# 情侣厨房 —— 信息架构

> 本文档定义页面层级、Tab 结构、跨页面入口和关键状态机。
> 配合 [功能清单](./02-feature-list.md) 和 [用户流程](./04-user-flows.md) 阅读。

---

## 1. 顶层 Tab 设计

MVP 采用 **4 Tab** 结构。原则：每个 Tab 只承担一类心智，不混合。

| Tab | 名称           | 主要心智            | 默认进入页              |
| :-: | -------------- | ------------------- | ----------------------- |
|  1  | 小厨房（首页） | "现在/今天发生什么" | 今日订单卡片 + 快速操作 |
|  2  | 菜单           | "我们家有什么菜"    | 家庭菜单库列表          |
|  3  | 时间线         | "我们一起吃过什么"  | 食物日记瀑布流          |
|  4  | 我的           | "我的账户与设置"    | 爱心币 + 成就 + 设置    |

> ⚠️ 不放"消息"独立 Tab：消息天然挂在订单详情页内，独立 Tab 会让用户分不清"消息"和"订单"。
> ⚠️ 不放"发现/推荐"Tab：MVP 不做内容池，避免空虚。

## 2. 页面树（二级 + 三级）

### Tab 1 · 小厨房（首页）

```
首页（小厨房）
├── [顶部] 家庭空间名 + 双人头像
├── [卡片区] 当前活跃订单卡片（如有）
│   └── 点击 → 订单详情页
├── [快捷入口] 我要点菜（→ 创建订单）
├── [快捷入口] 我来做菜（→ 待我接单列表）
└── [底部] 最近完成订单（3 条预览）→ 时间线
```

### Tab 2 · 菜单

```
菜单
├── [顶部搜索栏] 菜名搜索
├── [筛选 chips] 早/午/晚/夜宵 · 口味标签
├── [列表] 菜谱卡片（图 + 名 + 难度 + 被点过 N 次）
│   └── 点击 → 菜谱详情
│       ├── 大图轮播
│       ├── 标签 / 难度 / 备注
│       ├── 历史评分
│       └── [按钮] 点这道（→ 创建订单，预填）
├── [浮动按钮] + 新建菜谱
│   └── 新建 / 编辑菜谱页
└── [子页] 收藏菜（P1）
```

### Tab 3 · 时间线

```
时间线
├── [顶部] 月份切换器
├── [瀑布流] 时间线卡片
│   └── 关联订单卡：菜品图 + 双方留言 + 评分 + 当日爱心币
├── [点击卡片] → 条目详情页
│   ├── 完整信息
│   └── [操作] 隐藏 / 删除条目（部分实现）
└── [入口] 返回月份列表
```

### Tab 4 · 我的

```
我的
├── [顶部] 头像 + 昵称 + 厨师等级
├── [卡片] 爱心币余额 + 本月变动
│   └── 点击 → 爱心币流水页
├── [卡片] 成就预览（最近解锁的 3 个徽章）
│   └── 点击 → 成就墙
├── [卡片] 数据统计：累计做菜数、连续做饭天数
├── [菜单] 家庭空间设置
│   ├── 改名 / 改纪念日
│   ├── 成员列表
│   ├── 邀请新成员（P2 多人）
│   └── 退出家庭空间
├── [菜单] 个人资料
├── [菜单] 意见反馈
├── [菜单] 关于
└── [菜单] 注销账号
```

## 3. 非 Tab 主要页面

这些页面通过其他页面跳转进入，没有 Tab 入口。

### 3.1 订单相关

| 页面         | 进入路径                        | 主要内容                        |
| ------------ | ------------------------------- | ------------------------------- |
| 创建订单页   | 首页快捷入口 / 菜谱详情"点这道" | 选厨师 + 选菜 + 备注 + 期望时间 |
| 待接单列表   | 首页"我来做菜" / 通知点击       | 厨师视角，待我处理的订单        |
| 订单详情页   | 多入口（见 §4）                 | 状态机 + 消息流 + 操作按钮      |
| 历史订单列表 | 我的 / 时间线"查看更多"         | 全量历史，可筛选                |

### 3.2 沟通相关

| 页面         | 进入路径                     | 备注                   |
| ------------ | ---------------------------- | ---------------------- |
| 订单内消息流 | 嵌入订单详情页（不是独立页） | 含表情包面板、催菜按钮 |

### 3.3 爱心币 / 成就

| 页面           | 进入路径              | 备注                  |
| -------------- | --------------------- | --------------------- |
| 爱心币流水     | 我的 → 爱心币卡片     | 个人 + 家庭双账本切换 |
| 成就墙         | 我的 → 成就卡片       | 全部徽章网格          |

### 3.4 系统页

- 登录 / 首次引导
- 创建 / 加入家庭空间
- 邀请等待页（已发出邀请，等对方接受）
- 隐私协议、用户协议、版本更新页
- 反馈页（用户提交意见反馈）
- 关于页（版本号、更新日志）
- 绑定凭证页（管理后台登录）

## 4. 跨页面入口（一图多路径）

订单详情是核心交汇页，可从多处进入：

```
                  [首页活跃订单卡]
                          │
[通知点击] ────────────► [订单详情] ◄──────── [菜谱详情→点这道→新订单→详情]
                          ▲
                          │
                  [时间线条目卡]
                          ▲
                          │
                  [历史订单列表]
```

设计含义：用户在任何场景下想"看这单怎么样了"，都能 1–2 步到达。

## 5. 关键状态机

### 5.1 订单状态机（详细版）

```
       ┌──────────┐  创建    ┌──────────┐
       │   草稿   │ ──────► │ 待接单   │
       └──────────┘          └────┬─────┘
                                  │
                  ┌───────────────┼──────────────────┐
                  ▼               ▼                  ▼
            [接单]            [拒单]            [取消(食客)]
                  │               │                  │
                  ▼               ▼                  ▼
            ┌──────────┐    ┌──────────┐    ┌──────────┐
            │ 已接单   │    │ 已拒绝   │    │ 已取消   │
            └────┬─────┘    └──────────┘    └──────────┘
                 │
        [厨师切换] 备菜中 → 烹饪中
                 │
                 ▼
            ┌──────────┐  上菜    ┌──────────┐
            │ 烹饪中   │ ───────► │ 已上菜   │
            └──────────┘          └────┬─────┘
                                       │ 评价
                                       ▼
                                 ┌──────────┐
                                 │ 已评价   │
                                 └────┬─────┘
                                      │ 自动 / 24h
                                      ▼
                                 ┌──────────┐
                                 │ 已完成   │
                                 └──────────┘
```

**关键约束**：

- 同一家庭同时只允许 1 个"非终态"订单（草稿除外）
- 已上菜后不可取消
- 已评价后自动转已完成（或 24h 后自动转），用于生成时间线条目并发放爱心币

### 5.2 家庭空间状态机

```
[空] → [我已创建,等对方加入] → [双人已绑定] → [单人退出] → dissolving →定时任务→ [dissolved（软删除，数据保留）]
                                       │
                                       └─► [对方主动加入] = 稳态
```

## 7. 命名约定（避免文档间术语漂移）

| 概念               | 统一称呼          | 不要叫成           |
| ------------------ | ----------------- | ------------------ |
| 两人共享的工作区   | 家庭空间 / 小厨房 | 群组、组、伴侣空间 |
| 下单的一方         | 食客              | 顾客、买家、客人   |
| 做菜的一方         | 厨师              | 大厨、卖家、店主   |
| 一笔点菜请求       | 订单              | 单子、点单、请求   |
| 单笔订单内的对话   | 订单消息 / 沟通   | 聊天、对话         |
| 虚拟积分           | 爱心币            | 积分、金币、糖糖   |
| 自动生成的回忆条目 | 时间线条目        | 日记、日志、动态   |

所有 UI 文案、文档、变量名（英文化时）务必沿用此命名。
